02

How it’s cracked

Ghost Font scrolls the noise dots inside the glyph mask upward while the background dots scroll downward. Because the encoder never special-cases the boundary, that design leaks the letters into every single frame.

1

Shared lattice, split motion

The two dot layers share a horizontal grid phase and differ only vertically. Where they overlap they fuse into solid 3×4 / 3×5 / 3×6 blocks — pure interior noise that carries no edge.

2

Union-erase

Erase every pixel covered by any solid k×k dark block. All interior dots vanish; only the clipped fragments along the glyph boundary survive — and they trace the outline. No motion tracking, one frame.

3

Read the outline

A human can read the outline directly. For automation we match each glyph against outline templates of the known charset (A–Z, 0–9) with NCC, then vote across frames to shrug off compression noise.

The faint horizontal bands at the top and bottom are structural: dots entering and leaving the canvas leave partial blocks behind. They are not the decoy caption — that “WRITTEN IN GHOST FONT” layer sits at 0.025 opacity and vanishes at the binarization threshold, so it never reaches this pipeline at all.

03

The point

Ghost Font is an experiment in writing that humans read and AI can’t. As a piece of performance art, unghost completes the loop — not just by breaking it, but by teaching AI to break it.

Read it with an AI

This repo ships a SKILL.md that instructs a multimodal model to recover the outline and read it natively — which, for an AI, is often the most natural route. Classical template OCR is kept as the fallback. Point your agent at the skill and hand it a frame: 

# in Claude Code / any skill-aware agent
/skill unghost
# then: “read the hidden message in this clip”

An in-browser CNN recognizer is on the roadmap — the same idea, learned end-to-end.
